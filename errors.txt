Ein sauberer reaktiver Kreislauf sieht oft so aus:

    Parent setzt ein Attribut am Child (z. B. value="5").
    Child reagiert intern (z. B. zeigt eine „5“ an).
    Nutzer klickt im Child auf „Plus“.
    Child berechnet intern „6“, ändert aber nicht sein eigenes Attribut (Single Source of Truth!), sondern sendet ein Event (change, detail: 6).
    Parent empfängt das Event und entscheidet, ob es das Attribut des Childs nun auf „6“ aktualisiert.

<script>
  let count = $state(0); // Zentraler reaktiver State
</script>

<!-- Kein "on:change={...}", sondern eine einfache Prop -->
<Child value={count} onchange={(v) => count = v} />

Wenn der Tab sichtbar ist (auch wenn er nicht den Fokus hat), verhindert der Wake Lock, dass das System in den Energiesparmodus geht oder den CPU-Takt für diesen Tab massiv senkt.


let wakeLock = null;
async function requestLock() {
    if ('wakeLock' in navigator) {
        wakeLock = await navigator.wakeLock.request('screen');
    }
}
